FILE2PDF – Command Line Argument Summary
========================================

Basic usage
-----------
-command=file2pdf -input=FOLDER -output=FILE.pdf [options]

Example
-------
-command=file2pdf -input="D:\scans" -output="book.pdf" -pagesize=A4 -margin=10


Required Arguments
------------------

-command=file2pdf
    Runs the file → PDF conversion pipeline.

-input=folder
    Folder containing images and/or PDFs.

-output=file.pdf
    Output PDF file path.

Supported input file types
--------------------------
jpg
jpeg
png
bmp
gif
tif
tiff
webp
pdf

Files are processed in natural filename order:
1, 2, 3, 10 (not 1, 10, 2).


Layout Arguments
----------------

-pagesize
    Controls output page size.

    Examples
    -pagesize=A4
    -pagesize=A3
    -pagesize=A0
    -pagesize=auto
    -pagesize=1_1

    Behavior
    A5–2A0   → Force fixed ISO paper size
    auto     → Detect closest A-series paper
    1_1      → Page size = content size + margins
    omitted  → Uses source size


-fitwidth
    Scale pages so the short side equals this width (mm).

    Example
    -fitwidth=210

    Useful for
    • normalizing mixed scans
    • consistent book pages
    • printing layouts

    Overrides normal page sizing.


-margin
    Page margin in millimeters.

    Default
    8

    Example
    -margin=15


-stretch
    Allow stretching to fill the page.

    -stretch=y
    -stretch=n

    Behavior
    y → Fill page area
    n → Preserve aspect ratio

    Default
    y


-percentthreshold
    Used only when pagesize=auto.

    Defines tolerance when matching A-series paper.

    Example
    -percentthreshold=5

    Meaning
    5% size difference allowed

    Default
    5


Page Number Arguments
---------------------

-numberoffset
    Distance of page number from bottom-left corner (mm).

    Example
    -numberoffset=6

    Default
    4


Example Commands
----------------

Simple conversion
-command=file2pdf -input=images -output=out.pdf


Book scanning workflow
-command=file2pdf -input=scans -output=book.pdf -pagesize=A4 -margin=12 -stretch=n


Normalize mixed scans
-command=file2pdf -input=scans -output=normalized.pdf -fitwidth=210


Auto detect paper
-command=file2pdf -input=scans -output=print.pdf -pagesize=auto


Internal Processing Pipeline
----------------------------
Scan folder
↓
Sort filenames naturally
↓
Detect file type
↓
Image → layout engine
PDF   → layout engine
↓
Fit into page
↓
Add page number
↓
Write final PDF


Defaults (if not specified)
---------------------------

margin             = 8 mm
numberoffset       = 4 mm
stretch            = true
percentthreshold   = 5
pagesize           = A4
font               = Arial
fontsize           = 5


Typical Uses
------------
• scan-to-PDF workflows
• comic / manga conversion
• document normalization
• print preparation
• merging images and PDFs
